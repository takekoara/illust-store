# ポートフォリオ用プロジェクト説明

## このプロジェクトについて

このプロジェクトは、**フルスタックWebアプリケーション開発**のスキルを実証するために作成されたイラスト販売プラットフォームです。

## アピールポイント

### 1. モダンな技術スタック
- **Laravel 11**: 最新のPHPフレームワークを使用
- **React + TypeScript**: 型安全性を重視したフロントエンド開発
- **Inertia.js**: サーバーサイドとフロントエンドをシームレスに統合

### 2. 実践的な機能実装
- **決済統合**: Stripe APIを使用した本格的な決済機能
- **認証・認可**: Laravel Breeze、Policies、ミドルウェアによる多層的なセキュリティ
- **ファイルアップロード**: 複数画像のアップロードとドラッグ&ドロップ並び替え
- **メール通知**: 注文完了時の自動メール送信（Laravel Notificationsを使用。デモ環境ではログに記録、本番環境では実際に送信可能）

### 3. コード品質
- **TypeScript**: 型安全性によるバグの早期発見
- **Laravel Best Practices**: Eloquent ORM、Policies、Form Requestsの適切な使用
- **コンポーネント設計**: 再利用可能なReactコンポーネント設計
- **エラーハンドリング**: 適切な例外処理とエラーページ

### 4. ユーザー体験
- **レスポンシブデザイン**: モバイル・タブレット・デスクトップ対応
- **直感的なUI**: Tailwind CSSによるモダンなデザイン
- **画像ギャラリー**: モーダル表示、マウスホイールナビゲーション
- **日本語対応**: 完全な日本語ローカライゼーション

## 技術的なハイライト

### バックエンド
- **RESTful API設計**: 適切なHTTPメソッドとステータスコードの使用
- **データベース設計**: 正規化されたテーブル構造、リレーションシップ
- **セキュリティ**: CSRF保護、XSS対策、SQLインジェクション対策
- **非同期処理**: キューを使用したメール送信（本番環境対応。デモ環境ではログドライバーで動作確認可能）

### フロントエンド
- **コンポーネント設計**: 再利用可能で保守しやすいコンポーネント
- **状態管理**: Inertia.jsによるサーバー状態管理
- **型安全性**: TypeScriptによる型定義
- **アクセシビリティ**: Headless UIによるアクセシブルなコンポーネント

### 統合
- **Stripe Webhook**: 非同期での決済完了処理
- **ファイルストレージ**: Laravel Storageによるファイル管理
- **環境変数管理**: セキュアな設定管理

## デモ・スクリーンショット

### 推奨されるスクリーンショット
1. **ダッシュボード**: 統計情報の表示
2. **商品一覧**: グリッドレイアウト
3. **商品詳細**: 画像ギャラリー、モーダル表示
4. **カート**: ショッピングカート機能
5. **決済画面**: Stripe決済フォーム
6. **注文履歴**: 注文一覧と詳細

## 面接での説明ポイント

### 「なぜこの技術スタックを選んだか？」
- **Laravel**: 豊富な機能、優れたドキュメント、コミュニティサポート
- **React + TypeScript**: 型安全性、再利用性、モダンな開発体験
- **Inertia.js**: SPAの体験とサーバーサイドレンダリングの利点を両立

### 「最も難しかった部分は？」
- **Stripe統合**: Webhookの処理、Payment Intentの管理
- **画像アップロード**: 複数画像の処理、ドラッグ&ドロップ実装
- **認可制御**: 管理者権限の適切な実装

### 「改善したい点は？」
- テストカバレッジの向上
- パフォーマンス最適化（画像最適化、キャッシュ）
- リアルタイム機能の実装（WebSocket）
